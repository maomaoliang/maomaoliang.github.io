## 什么是流式输出

流式输出（Streaming）是指服务器在生成完整响应之前，就已经开始逐步返回数据给客户端。与传统的一次性返回整个响应不同，流式输出让用户能够"实时"看到AI生成的内容，像打字机一样逐字逐句呈现。

## 非流式输出

非流式输出是最传统的HTTP响应方式：

```
客户端 → 请求 → 服务器
                ↓
           等待生成完整响应
                ↓
客户端 ← 完整响应 ← 服务器
```

**特点**：
- 服务器必须等待整个响应生成完毕后才返回
- HTTP响应头包含 `Content-Length` 标明完整长度
- 必须等AI跑完整段代码或文章才看到第一个字

**缺点**：首字延迟等于输出总时长，体验差。

## 流式输出

流式输出的流程是这样的：

```
时间线 →
────────────────────────────────────────────────────→

服务器:   [生成]→[chunk1]→[生成]→[chunk2]→[生成]→[chunk3]→[DONE]
              ↓          ↓          ↓          ↓
客户端:   [显示1]  [显示12]   [显示123]  [显示1234] ...
```

**原理**：
- HTTP分块传输编码：`Transfer-Encoding: chunked`
- 服务器**生成一点就返回一点**，不需要等全部完成
- SSE（Server-Sent Events）或WebSocket实时推送

**优点**：
- 首字延迟低（几百毫秒）
- 用户感知"打字机"效果
- 实时性好

## 两者对比

| 指标 | 非流式 | 流式 |
|------|--------|------|
| 首字延迟 | 高（等全部生成） | 低（几十~几百ms） |
| 实时性 | 差 | 好 |
| 实现难度 | 简单 | 稍复杂 |
| 适用场景 | 小响应、短文本 | AI对话、长回答 |

## 前端实现示例

### 流式输出（SSE）

```javascript
const response = await fetch('/api/chat', {
  method: 'POST',
  body: JSON.stringify({ message: '你好' })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const text = decoder.decode(value);
  // 逐块追加到界面
  outputDiv.innerHTML += text;
}
```

### 非流式输出

直接 `await response.json()` 获取完整JSON，简单但等待时间长。

## 总结

流式输出就像**自助餐排队**——厨师做好一道上一道；而非流式像**端一盘菜上桌**——全部做好才端出来。

现在主流的AI对话产品（如ChatGPT、Claude、CodeBuddy）都采用流式输出，为的就是让用户能够实时看到AI"思考"和"打字"的过程，提升交互体验。
